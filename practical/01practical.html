<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>DEEPAK</title>
</head>
<body>
<h3> My Name Is Deepak Prajapati. </h3>
</body>
</html>

OUTPUT:

My Name Is Deepak Prajapati.

DOCTYPE Declaration (<!DOCTYPE html>):

This declaration defines the type of document being used. In this case, we're using the HTML5 doctype. It helps the browser understand which version of HTML is being used in the document. Using the correct doctype ensures that the browser renders the webpage in standards-compliant mode, preventing compatibility issues and ensuring consistent rendering across different browsers.

Meta Tags:

1. <meta charset="UTF-8">

This meta tag specifies the character encoding for the document. UTF-8 is a character encoding that supports a wide range of characters from different languages. It ensures that the text on the webpage is displayed correctly, regardless of the language used.

2. <meta name="description" content="Webpage displaying my name>

This meta tag provides a brief description of the webpage's content. Search engines often use this description in search results to give users a preview of the page's content. It's important for search engine optimization (SEO) and for helping users understand the purpose of the webpage.

3. <meta name="keywords" content="name, example, web page">

Although not as crucial for SEO as it once was, this meta tag used to provide a list of keywords related to the content of the webpage. Search engines would use these keywords to help index and categorize the webpage. However, modern search engines primarily focus on the actual content of the page rather than relying heavily on this meta tag.

4. <meta name="author" content="ChatGPT">

This meta tag indicates the author of the webpage's content. It can be useful for attributing content to its creator, and it might also provide a way for users to contact the author if needed.
